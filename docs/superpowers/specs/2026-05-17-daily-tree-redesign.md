**日期:** 2026-05-17  
**版本:** 1.0 - 待执行

## 一、项目概述

Daily Tree 是一个每日记录应用，通过"种树"的隐喻让用户坚持日常记录。当前版本使用抽象的同心圆设计展现树的成长。本次重设计的目标是：

1. **树的视觉真实化** — 从抽象圆圈升级到高逼真树形（有树干、树杈、根系）
2. **生长过程可感知** — 树从种子开始，通过用户的连续记录逐渐成长，在特定里程碑（7/15/30/180/365天）有明显变化
3. **四季变化和天气系统** — 树的外观随季节和实时天气变化，增加"活着"的感觉
4. **完整的打卡功能** — 支持日期选择、年份切换、当天编辑删改
5. **交互提升** — 树可拖动旋转，卡片可左右滑动，整体交互更流畅

---

## 二、设计方向

### 2.1 树的风格：高逼真

树的外观应该接近真实树木，而非几何抽象。包含以下元素：
- **树干** — 真实的树干纹理和形状
- **树杈** — 可见的分支结构
- **树冠** — 饱满的叶片区域
- **根系** — 地下根部的呈现
- **季节变化** — 春绿、夏深、秋金、冬白

### 2.2 色彩和质感

保持现有的设计系统（深色沉稳），但通过动画和效果增加生命力：
- **深色背景** — `#0A0A0B`（Void Black）保持不变
- **树的核心色** — `#4A7C59`（Canopy Green）为基调
- **季节色变** — 春粉红、夏深绿、秋金黄、冬苍白
- **天气背景** — 晴（明亮）/ 阴（柔和）/ 雨（青绿）/ 雪（冷白）

### 2.3 英文字体升级

Landing page 的英文标题升级为高端衬线体（如 Fraunces），提升视觉优雅感。应用内保持 Outfit 现代字体。

### 3.2 三个阶段卡片 (Deck A)

展示用户行为的三个阶段，每个阶段配对应状态的树：

| 卡片 | 标题 | 描述 | 树的样子 |
|------|------|------|---------|
| 1 | 日记 | 每一个记录都是树的养分 | 幼苗期（较小，枝叶稀疏） |
| 2 | 月总结 | 一个月的坚持见证树的成长 | 生长期（中等大小，分支清晰） |
| 3 | 年成长 | 一年的坚持，树已成材 | 成熟期（饱满茂密，老练） |

**交互方式：** 用户点击卡片切换，当前选中的卡片高亮，下方实时显示对应的树（可用 Three.js 或 SVG 动画）。

### 3.3 四季卡片 (Deck B)

展示树在四季的不同样貌（同一棵树的不同季节状态）：

| 卡片 | 季节 | 视觉效果 |
|------|------|---------|
| 1 | 春 | 淡绿色 + 粉色花朵粒子 |
| 2 | 夏 | 深绿色，最茂密 |
| 3 | 秋 | 金黄色 + 落叶粒子 |
| 4 | 冬 | 苍白色，少叶子 + 雪粒子 |

**交互方式：** 同上。可选择自动循环播放（每 3 秒切换一个季节）或用户手动切换。

### 3.4 响应式设计

### 4.2 树的视觉和成长规则

#### 视角：侧面视图（Side View）
从树的侧边观看，能清楚看到：
- 树干的形状和纹理
- 树冠的轮廓和分层
- 根系的分布

#### 成长规则：按连续记录天数

树从**种子**开始，每天一条记录导致一次微小变化，在特定里程碑有**明显的形态变化**：

| 天数 | 状态 | 视觉描述 |
|------|------|---------|
| 0 | 种子 | 一颗小种子，埋在地里 |
| 1-6 | 萌芽期 | 露出地面的小芽，细细的 |
| 7 ⭐ | 小植株 | 出现第一对叶子，可见简单的主干 |
| 8-14 | 幼苗期 | 更多叶子，主干变粗 |
| 15 ⭐ | 小树 | 开始出现分支，树冠形成 |
| 16-29 | 成长期 | 树的分支增多，树冠更饱满 |
| 30 ⭐ | 小成树 | 明显的树干和树冠分离，根系可见 |
| 31-179 | 稳定生长期 | 树冠和树干的比例逐渐优化，树变得更真实 |
| 180 ⭐ | 成树 | 树形成熟，所有细节清晰可见（树干、主要分支、根系） |
| 181-364 | 完善期 | 树的细节和纹理进一步丰富 |
| 365 ⭐ | 古树 | 树达到最终形态，树干粗壮，分支复杂，根系深实 |

**里程碑处理：** 在第 7/15/30/180/365 天，树应该有一个明显的、可被用户感知的变化（不是渐进式的，而是一次跳变）。可配合动画（树快速长大、树干加粗等）和通知（"恭喜坚持 X 天！"）。

#### 季节影响（当前年的树）

树的外观同时受**季节**影响（基于当前日期的月份）：

| 季节 | 月份 | 树的颜色 | 特殊效果 |
|------|------|--------|---------|
| 春 | 3-5 | 淡绿 `#7DB776` | 粉色花朵粒子在树冠顶端飘落 |
| 夏 | 6-8 | 深绿 `#4A7C59` | 无特殊粒子，树最茂密 |
| 秋 | 9-11 | 金黄 `#D4A574` | 金黄色叶子粒子从树冠飘落 |
| 冬 | 12-2 | 苍白 `#A9A9A9` | 白色雪花粒子，树变稀疏（叶子减少） |

**实现方式：** 使用 Three.js 的材料颜色 uniform + 粒子系统，根据 `new Date().getMonth()` 实时计算。

#### 过往年份的树

显示过往年份（2024、2025等）的树时，保持该年份的最终形态（Day 365或该年的最后一条记录时的形态），不再随季节变化。

### 4.3 天气背景系统

#### 数据源
- 获取用户当前位置的实时天气（需要位置权限）
- 使用公共天气 API（如 OpenWeatherMap、WeatherAPI）
- 缓存天气数据 1 小时，减少 API 调用

#### 天气类型和视觉效果

| 天气 | 图标 | 背景效果 | 色调 |
|------|------|---------|------|
| 晴天 ☀️ | 太阳 | 明亮的径向渐变，从上方倾泻光线 | 温暖黄色 + 深蓝天空 |
| 阴天 ☁️ | 云朵 | 柔和的灰色渐变，光线柔和 | 冷中性 |
| 雨天 🌧️ | 雨滴 | 动画化的雨线，略微青绿色调 | 青绿 + 深灰 |
| 雪天 ❄️ | 雪花 | 动画化的雪花飘落，静谧感 | 冷白 + 深蓝 |

**实现方式：** 使用 Three.js 的动态材料和粒子系统。背景与树的季节色不冲突，背景是环境光，树是主体。

### 4.4 打卡和日期选择

#### 功能需求
1. 用户点击 `+` 按钮打开新增记录 modal
2. Modal 显示当前选中的**年份**和**当天日期**
3. 用户**不能**补录过去的日期，只能记录当天
4. 用户可以选择**其他年份**（通过 Dock 的年份选择器切换）
5. 写入内容后提交，树增加一天的成长，新卡片出现在底部

#### Modal 设计
```
┌─────────────────────────┐
│ 2026 | 2026-05-17      │  <- 年份 + 当天日期（只读）
│ What matters most today? │  <- 提示文案
├─────────────────────────┤
│                          │
│  [大文本框，输入日记]    │
│                          │
├─────────────────────────┤
│            [提交]        │  <- 右下角提交按钮
└─────────────────────────┘
```

#### 年份切换
- Dock 左侧显示过往年份的小树缩略图 + 当前年份指示器
- 用户点击某个年份，切换到该年份的树和记录
- 该年份的卡片在底部显示

### 4.5 卡片和当天记录交互

#### 卡片展示
- 底部展示**当天的所有记录**（按时间顺序）
- 每张卡片显示日期、时间（可选）、内容预览
- 卡片宽度占满屏幕，可**左右滑动**翻页查看其他卡片

#### 当天编辑权限
- **当天的记录** — 可点击编辑、可删除
- **往日的记录** — 只读（展示，但不可编辑/删除）

#### 卡片交互
1. **点击卡片** — 展开全文或进入详情页（可编辑）
2. **左右滑动** — 翻页到其他卡片
3. **长按** — 显示删除选项（仅当天记录）

### 4.6 树的交互

#### 拖动旋转
- 用户在树上拖动（手指/鼠标），树围绕 Y 轴旋转
- 旋转是平滑的，阻尼衰减（松开后自动停止）
- 旋转范围：可 360° 旋转，无限制

#### 自动旋转（可选）
- 如果用户长时间不交互（如 10 秒），树可缓慢自动旋转（可选实现）

#### 缩放（可选）
- 点击树可能放大查看细节，或支持双指缩放（可后期加）

### 4.7 Chip 和状态指示

#### Streak Chip（顶部右侧）
- 显示当前连续记录天数
- 格式：`🔥 7 days` / `🔥 7天`
- 如果昨天没有记录，Streak 重置为 0（显示灰色 ⭕ 0）
- 每当新增一条记录，立即更新

#### Today Chip（顶部）
- 显示今天是否已记录
- 格式：`⭕ Today's tree hasn't grown yet.` / `✓ Today's entry recorded.`
- 样式变化：未记录时暗淡，已记录时高亮

## 五、技术考量

### 5.1 树的 3D 建模

**当前方案：** 使用 Three.js 的 `TubeGeometry` + `InstancedMesh`
- 树干：单条 TubeGeometry，沿着曲线生长
- 树杈：多条 TubeGeometry 从树干分支出来
- 树叶：InstancedMesh 的球体/椭球体，通过高斯分布在树冠周围
- 根系：3-4 条 TubeGeometry 从树干底部分支

**成长动画：** 使用顶点动画或形变动画，在里程碑时快速改变树的形状。

**季节变化：** 通过修改材料的颜色 uniform，以及控制叶子粒子的数量和透明度。

### 5.2 天气背景

**实现方式：** Three.js 的动态背景（使用 Canvas 或 ShaderMaterial）
- **晴天** — 渐变背景 + 阳光光线（rays）
- **阴天** — 均匀柔和的灰色
- **雨天** — 雨线粒子系统
- **雪天** — 雪花粒子系统

### 5.3 性能考虑

- 树的复杂度根据日数动态调整（Day 0 简单，Day 365 复杂）
- 粒子系统使用 GPU 加速（WebGL）
- 天气背景缓存，切换天气时平滑过渡（2-3 秒淡入淡出）
