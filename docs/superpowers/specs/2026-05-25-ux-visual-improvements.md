## 2. Landing Page — 引导卡片 UX 重构（方案 B）

**问题：** 现有三张可滑动哲学卡片挡在「进入 App」路径前，对着急使用的用户造成摩擦。

**方案：** 大 CTA 优先 + 折叠理念区

**布局变化：**
- 移除卡片作为页面主视觉区域
- 页面中央改为 hero 文字：「你的树，从今天开始生长」+ 副标题
- 下方紧跟大尺寸「立即开始」按钮（主转化路径）
- 按钮下方折叠区「💡 了解产品理念」，点击展开原三张卡片内容（保持滑动交互）

**行为：**
- 默认折叠，不阻碍进入
- 折叠区点击展开/收起，内容与现有三张卡片一致

## 3. App 引导页 — 第三步树形修复

**位置：** `app/index.html` `#guide-step-2`  
**问题：** 当前 step-2 使用手写 SVG 树形，与实际 tree.js 渲染的 7 阶段树完全不符。

**方案：** 用真实 canvas 替换 SVG

- 在 `#guide-step-2` 内用 `<canvas id="guide-tree-canvas-2">` 替换现有 SVG

- 保留「点击树枝，回顾过去」文字说明

## 5. 底部导航「树」按钮 — 成长阶段预览面板（方案 C）

**触发：** 点击底部导航栏「树」图标（`#btn-tree`）

**面板结构（bottom sheet）：**

```
┌──────────────────────────┐
│  ▬  (拖拽条)              │
│  树的一生                 │
├──────────────────────────┤
│  当前阶段                 │
│  ┌──────┐  青树           │
│  │canvas│  枝条开始分叉，  │
│  │young │  树冠逐渐成型。  │
│  └──────┘                │
│                           │
│  再 12 天 → 壮年树        │
│  [进度条]                 │
│                           │
│  🌰  🌱  🪴  [🌿]  🌳  🌲  │
└──────────────────────────┘
```

**各阶段数据：**

| 阶段 | key | 解锁天数 | canvas渲染天数 | 简介 |
|------|-----|---------|--------------|------|
| 种子 | seed | 1 | 1 | 万物始于一粒种子 |
| 嫩芽 | sprout | 3 | 3 | 破土而出，生命开始 |
| 幼苗 | sapling | 7 | 15 | 第一片叶，扎根成形 |
| 青树 | young | 30 | 50 | 枝条分叉，树冠成型 |
| 壮年 | mature | 90 | 180 | 繁茂深根，见证岁月 |
| 古树 | ancient | 300 | 400 | 历经风雨，屹立不倒 |

**实现要点：**
- 调用 `drawTree(canvas, stageMidpointDays, fixedSeed=12345)` 渲染当前阶段树形
- 读取 `dayCount` 计算「再 X 天」倒计时
- 底部缩略路径 emoji：已解锁高亮，未解锁灰暗半透明
- 点击背景关闭面板
